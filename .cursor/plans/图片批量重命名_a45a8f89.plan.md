# 图片批量重命名（1, 1-1, 2, 2-2…）

## 背景

项目为单文件 Tkinter 应用 [`main.py`](d:\code\alexcard_tools\main.py)，已有「长宽比 / 转为 JPG / 调整亮度」三个批量功能，均遵循 **纯函数 + 工具栏按钮 + 文本区报告 + 完成弹窗** 模式。

你已确认：**在原文件夹内直接重命名**，不复制到输出目录。

## 命名规则

按用户**选择顺序**（`askopenfilenames` 返回顺序）从 0 开始编号：

| 序号 (0-based) | 新文件名（不含扩展名） |
|----------------|------------------------|
| 0 | `1` |
| 1 | `1-1` |
| 2 | `2` |
| 3 | `2-2` |
| 4 | `3` |
| 5 | `3-3` |
| … | … |

公式：

```python
def rename_stem(index: int) -> str:
    n = index // 2 + 1
    return str(n) if index % 2 == 0 else f"{n}-{n}"
```

每张图片**保留原扩展名**（如 `.jpg`、`.png`）。

奇数张时最后一张仅得 `N`（无对应的 `N-N`），属预期行为。

```mermaid
flowchart LR
    clickBtn[点击批量重命名] --> pickFiles[多选图片]
    pickFiles --> validate[校验同目录与冲突]
    validate --> confirm[确认弹窗预览命名]
    confirm --> phase1[阶段1: 改为临时名]
    phase1 --> phase2[阶段2: 改为最终名]
    phase2 --> showReport[文本区显示结果]
    phase2 --> doneMsg[弹窗汇总成功失败]
```

## 实现方案

### 1. 新增纯函数（放在 `build_brightness_report` 之后）

**`rename_stem(index: int) -> str`**

- 上述命名公式，单独抽出便于测试与阅读。

**`rename_target_path(src_path: str, index: int) -> str`**

- `os.path.join(os.path.dirname(src_path), rename_stem(index) + ext)`
- `ext = os.path.splitext(src_path)[1]`（保留大小写，如 `.JPG`）

**`validate_rename_batch(paths: list[str]) -> str | None`**

- 空列表：返回错误信息
- **必须同一目录**：`os.path.dirname(p)` 全部相同，否则返回 `"所选图片须在同一文件夹内。"`
- **目标名冲突预检**：对每个 `(src, dest)`，若 `dest` 已存在且 `dest` 不在本次 `paths` 集合中，返回具体冲突文件名
- 若 `src == dest`（已是目标名），允许通过（后续跳过或记为成功）

**`rename_one_file(src: str, dest: str) -> tuple[bool, str]`**

- `os.rename(src, dest)`；成功返回 `"旧名 -> 新名"`，失败返回 `"旧名 - 错误: {e}"`

**`rename_images_batch(paths: list[str]) -> tuple[str, int, int]`**

核心逻辑（两阶段重命名，避免互相覆盖）：

1. 调用 `validate_rename_batch`，失败则返回单行错误报告 `(msg, 0, len(paths))`
2. 构建 `plan: list[tuple[src, dest]]`
3. **阶段 1**：每张 `src` → `{dir}/.__rename_tmp_{i}{ext}`（隐藏式临时名，`i` 为序号）
4. **阶段 2**：每个临时文件 → 对应 `dest`
5. 若阶段 2 中某步失败，报告已完成的项并标记失败（阶段 1 已完成时文件仍在临时名，用户可手动恢复；此边界在报告中说明即可，无需回滚逻辑）

跳过 `src == dest` 的项，报告为 `"xxx - 已是目标名，跳过"`。

**`build_rename_report(paths: list[str]) -> tuple[str, int, int]`**

- 封装 `rename_images_batch`，返回 `(report_text, ok_count, fail_count)`，与 JPG/亮度功能一致。

### 2. UI 改动

在 [`App.__init__`](d:\code\alexcard_tools\main.py) 工具栏「调整亮度…」旁增加：

```python
tk.Button(bar, text="批量重命名…", command=self.on_rename_images).pack(side=tk.LEFT, padx=(6, 0))
```

**`on_rename_images()` 流程：**

1. `filedialog.askopenfilenames(title="选择要重命名的图片", filetypes=IMAGE_FILETYPES)` — 无选择则返回
2. `paths = list(paths)` 保持选择顺序
3. 调用 `validate_rename_batch(paths)` — 若返回错误，`messagebox.showerror` 并返回
4. 生成预览文本（每行 `原文件名 -> 新文件名`），`messagebox.askyesno("批量重命名", preview)` — 取消则返回
5. 调用 `build_rename_report(paths)`
6. 结果写入 `self.text`，`messagebox.showinfo` 汇总成功/失败数

预览示例（4 张图）：

```
a.png -> 1.png
b.png -> 1-1.png
c.png -> 2.png
d.png -> 2-2.png
```

### 3. 边界情况

| 场景 | 处理方式 |
|------|----------|
| 跨目录多选 | 校验失败，弹窗提示 |
| 目标名已被其他文件占用 | 预检失败，不执行任何 rename |
| 源文件已是目标名 | 跳过，计为成功 |
| 奇数张图片 | 最后一张命名为 `N.ext`，确认弹窗中正常展示 |
| 不同扩展名混选 | 各自保留扩展名 |
| 单张失败 | 记录错误，继续处理其余（阶段 2 内） |
| 大小写冲突 (Windows) | 预检时 `os.path.normcase` 比较路径 |

### 4. 无需改动

- [`requirements.txt`](d:\code\alexcard_tools\requirements.txt)：仅用标准库 `os.rename`
- PyInstaller 打包配置：入口仍为 `main.py`
