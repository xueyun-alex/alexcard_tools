# 图片转 JPG 功能实现计划

## 背景

当前项目为单文件桌面应用 [`main.py`](d:\code\alexcard_tools\main.py)，已有「选择图片 → 查看长宽比 → 复制」流程，依赖 **Pillow** 读取图片。尚无格式转换逻辑，也无 `.save()` 调用。

你已确认：
- **UI**：在现有窗口增加按钮，与长宽比功能并存
- **输出**：每次转换前由用户选择输出文件夹

## 实现方案

```mermaid
flowchart LR
    clickBtn[点击转为JPG] --> pickFiles[多选图片文件]
    pickFiles --> pickDir[选择输出文件夹]
    pickDir --> batchConvert[逐张转换]
    batchConvert --> showReport[文本区显示结果]
    batchConvert --> doneMsg[弹窗汇总成功/失败数]
```

### 1. 新增转换逻辑（纯函数，与 UI 解耦）

在 [`main.py`](d:\code\alexcard_tools\main.py) 中、`build_report()` 附近添加：

**`jpg_output_path(src_path, out_dir) -> str`**
- 取源文件 basename，去掉扩展名，拼接 `{out_dir}/{name}.jpg`

**`prepare_for_jpeg(im: Image.Image) -> Image.Image`**
- JPEG 不支持透明通道，需统一转为 RGB：
  - `RGBA` / `LA`：白底合成（避免透明变黑色）
  - `P`（调色板，含透明 GIF/PNG）：先转 `RGBA` 再白底合成
  - 其他模式（如 `L`、`CMYK`）：`im.convert("RGB")`

**`convert_to_jpg(src_path, out_dir, quality=90) -> str`**
- 打开图片 → `prepare_for_jpeg` → `im.save(dest, "JPEG", quality=90)`
- 成功返回 `"文件名.jpg - 已保存 (宽×高)"`
- 失败返回 `"文件名 - 错误: {e}"`（与现有 `line_for_path` 错误风格一致）
- **GIF 动图**：Pillow 默认只读第一帧（符合 JPG 单帧特性，无需额外依赖）

**`build_convert_report(paths, out_dir) -> str`**
- 批量调用 `convert_to_jpg`，用换行拼接结果（复用现有 `build_report` 模式）

### 2. UI 改动

在 [`App`](d:\code\alexcard_tools\main.py) 工具栏中，「复制全部」旁增加按钮：

```python
tk.Button(bar, text="转为 JPG…", command=self.on_convert_to_jpg).pack(side=tk.LEFT, padx=(6, 0))
```

**`on_convert_to_jpg()` 流程：**
1. `filedialog.askopenfilenames(title="选择要转换的图片", filetypes=IMAGE_FILETYPES)` — 无选择则返回
2. `filedialog.askdirectory(title="选择 JPG 输出文件夹")` — 无选择则返回
3. 调用 `build_convert_report(paths, out_dir)`
4. 将结果写入 `self.text`（清空后插入，与「选择图片」行为一致）
5. `messagebox.showinfo` 汇总：成功 N 张、失败 M 张

### 3. 边界情况处理

| 场景 | 处理方式 |
|------|----------|
| 源文件已是 `.jpg` | 仍转换并写入输出目录（重新编码，非简单复制） |
| 不同目录下同名文件 | 输出目录中后者覆盖前者；结果文本中仍逐条记录 |
| 输出目录不存在 | `os.makedirs(out_dir, exist_ok=True)` 确保可写 |
| 单张失败 | 捕获异常，继续处理其余文件 |

### 4. 无需改动的部分

- [`requirements.txt`](d:\code\alexcard_tools\requirements.txt)：Pillow 已满足需求
- PyInstaller 打包（[`build_exe.bat`](d:\code\alexcard_tools\build_exe.bat)、[`ImgAspectRatio.spec`](d:\code\alexcard_tools\ImgAspectRatio.spec)）：入口仍为 `main.py`，无需修改

## 核心代码参考（新增部分示意）

```python
def prepare_for_jpeg(im: Image.Image) -> Image.Image:
    if im.mode in ("RGBA", "LA"):
        bg = Image.new("RGB", im.size, (255, 255, 255))
        bg.paste(im, mask=im.split()[-1])
        return bg
    if im.mode == "P":
        return prepare_for_jpeg(im.convert("RGBA"))
    if im.mode != "RGB":
        return im.convert("RGB")
    return im

def convert_to_jpg(src_path: str, out_dir: str, quality: int = 90) -> tuple[bool, str]:
    name = os.path.basename(src_path)
    dest = jpg_output_path(src_path, out_dir)
    try:
        with Image.open(src_path) as im:
            rgb = prepare_for_jpeg(im)
            os.makedirs(out_dir, exist_ok=True)
            rgb.save(dest, "JPEG", quality=quality)
            w, h = rgb.size
        return True, f"{os.path.basename(dest)} - 已保存 ({w}×{h})"
    except Exception as e:
        return False, f"{name} - 错误: {e}"
```
