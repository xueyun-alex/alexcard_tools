# 序号批量重命名（新增按钮）

## 现状

[`main.py`](d:\code\alexcard_tools\main.py) 已有「批量重命名…」按钮，规则固定为 `1, 1-1, 2, 2-2…`（`rename_stem` 用 `index // 2` 交替）。用户需要**另一套**规则，并通过**输入起始名**决定命名模式，因此新增独立按钮，不改动旧逻辑。

## 命名规则

按 `askopenfilenames` **选择顺序**，从 0 开始递增：

| 用户输入 | 第 1 张 | 第 2 张 | 第 3 张 | … |
|----------|---------|---------|---------|---|
| `1` | `1` | `2` | `3` | 起始整数 + index |
| `5` | `5` | `6` | `7` | 同上 |
| `1-1` | `1-1` | `2-2` | `3-3` | `{n}-{n}`，n = 起始整数 + index |
| `3-3` | `3-3` | `4-4` | `5-5` | 同上 |

- 每张图**保留原扩展名**（如 `.jpg`）
- 重命名在**原目录**内完成
- 非法输入（空、非数字、`1-2` 等左右不等）弹窗提示，不执行

```mermaid
flowchart LR
    clickBtn[点击序号重命名] --> pickFiles[多选图片]
    pickFiles --> askInput[输入起始名 1 或 1-1]
    askInput --> parse[parse_rename_pattern]
    parse --> validate[校验同目录与冲突]
    validate --> confirm[预览确认弹窗]
    confirm --> rename[两阶段 os.rename]
    rename --> report[文本区 + 完成弹窗]
```

## 实现方案

### 1. 新增模式解析

在 `rename_stem` 附近添加：

```python
@dataclass(frozen=True)
class RenamePattern:
    mode: Literal["sequential", "paired"]
    start: int

def parse_rename_pattern(text: str) -> RenamePattern | None:
    # strip 后：
    # - 纯整数 → sequential, start=int
    # - 形如 "N-N" 且两侧整数相等 → paired, start=N
    # - 否则 → None
```

```python
def rename_stem_from_pattern(index: int, pattern: RenamePattern) -> str:
    n = pattern.start + index
    if pattern.mode == "sequential":
        return str(n)
    return f"{n}-{n}"
```

### 2. 轻量重构：复用校验与重命名管线

为避免复制 `validate_rename_batch` / `rename_images_batch` / 预览弹窗，将 **stem 生成** 抽成参数（旧逻辑不变）：

| 函数 | 改动 |
|------|------|
| `rename_target_path(src, index, stem_fn)` | 新增 `stem_fn: Callable[[int], str]` 参数 |
| `validate_rename_batch(paths, stem_fn)` | 同上 |
| `build_rename_preview(paths, stem_fn)` | 同上 |
| `ask_rename_confirm(parent, paths, stem_fn, title_hint?)` | 可选更新提示文案 |
| `rename_images_batch(paths, stem_fn)` | 同上 |
| `build_rename_report(paths, stem_fn)` | 同上 |

旧按钮调用处传入 `rename_stem`；新按钮传入 `lambda i: rename_stem_from_pattern(i, pattern)`。

**两阶段重命名**（`.__rename_tmp_{i}{ext}` → 最终名）与冲突预检（`os.path.normcase`）逻辑完全复用，无需重写。

### 3. UI：新增按钮与事件

工具栏在「批量重命名…」旁增加：

```python
tk.Button(bar, text="序号重命名…", command=self.on_rename_by_pattern).pack(...)
```

**`on_rename_by_pattern()` 流程：**

1. `filedialog.askopenfilenames` — 无选择则返回
2. `simpledialog.askstring("序号重命名", "起始名称（如 1 → 1,2,3；1-1 → 1-1,2-2,3-3）：", initialvalue="1")` — 取消则返回
3. `parse_rename_pattern` — 失败则 `messagebox.showerror`
4. `validate_rename_batch(paths, stem_fn)` — 失败则 error 弹窗
5. `ask_rename_confirm(...)` 展示 `原名 -> 新名` 预览 — 取消则返回
6. `build_rename_report(paths, stem_fn)` → 写入文本区 + `messagebox.showinfo` 汇总

预览示例（输入 `1`，3 张图）：

```
a.png -> 1.png
b.png -> 2.png
c.png -> 3.png
```

预览示例（输入 `1-1`，3 张图）：

```
a.png -> 1-1.png
b.png -> 2-2.png
c.png -> 3-3.png
```

### 4. 边界情况

| 场景 | 处理 |
|------|------|
| 跨目录多选 | 校验失败 |
| 目标名被目录内其他文件占用 | 预检失败，不执行任何 rename |
| 源文件已是目标名 | 跳过，计成功 |
| 不同扩展名混选 | 各自保留扩展名 |
| Windows 大小写冲突 | `normcase` 预检 |
| 输入 `1-2`、`abc`、空 | 解析失败，提示格式说明 |
